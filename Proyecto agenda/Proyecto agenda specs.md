Se propone la realización de una agenda-calendario.





Debe tener capacidad para asignar citas a uin servicio único (no se puede atender a más de un cliente simultáneamente desde el punto de vista del negocio).



* Por tanto evita solapamientos.
* Tiene una interfaz gráfica básica que es un formulario.
* Como complemento tiene interfaz gráfica para ver el modo calendario.
* Como complemento tiene la funcionalidad de al seleccionar un día de calendario ver todas las citas asignadas y horas libres.
* Una primera versión podría sólo considerar horas enteras de 10:00 a 20:00, siendo de una sola hora cada cita.
* En todas las versiones se permite que un usuario pueda tener múltiples citas.
* Otra consideración es que la aplicación está destinada al profesional, (no al usuario final) que a través de formulario de alta de citas.
* Se puede "inspirar" en código ya hecho para citas (agendas, appointment calendars...)



Para nota:

Que al pasar el ratón por los días aparezca la estructura de citas de ese día.



Se debería crear el formulario también.
