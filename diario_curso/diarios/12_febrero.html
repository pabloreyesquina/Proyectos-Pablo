1. Navegación y Rutas

Aprendimos a movernos entre archivos y dentro del mismo documento.

ConceptoDescripción / Ejemplo
Ruta Relativa../archivo.html (Sube un nivel). Clave para portabilidad.
Anclashref="#id-destino" Navega dentro de la misma página.
Atributostarget="_blank" (Nueva pestaña), title (Tooltip), download.

🚨 Nota sobre Rutas y Live Preview

Si el enlace ../ falla en VS Code, asegúrate de haber abierto la carpeta raíz del proyecto. Si solo abres la subcarpeta "diarios", el servidor no podrá acceder a los archivos de arriba.

2. Elementos Multimedia: <img>

Etiqueta de autocierre para insertar contenido visual.

  • src: Origen de la imagen (Ruta local o URL).
  • alt: Texto alternativo (Obligatorio por accesibilidad y SEO).
  • loading="lazy": Carga diferida para mejorar el rendimiento.

Formatos recomendados:

WebP (Moderno/Eficiente), SVG (Vectorial/Escalable), PNG (Transparencias) y JPG (Fotos).

Ejercicio Práctico: Imágenes clicables

Combinamos <a> e <img> para que una imagen actúe como botón:

<a href="grande.jpg" download="mi-foto">
    <img src="peque.jpg" alt="Descripción de la imagen">
</a>

3. Contenedores Genéricos

Elementos sin significado semántico pero vitales para el diseño (CSS).

<div> (Block)

Ocupa todo el ancho. Salto de línea automático. Ideal para secciones y estructuras.

<span> (Inline)

Ocupa solo su contenido. No hay salto de línea. Ideal para estilos dentro de párrafos.

🚨 Alerta de Examen: Semántica vs Genéricos

REGLA DE ORO: Si existe una etiqueta semántica (<header>, <nav>, <article>), úsala. Evita la "div-itis" (uso excesivo de divs). Los roles ARIA solo se usan si no hay etiqueta semántica disponible.
