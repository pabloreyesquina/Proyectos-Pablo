1. Etiquetas y Accesibilidad Inicial

Empezamos la clase repasando la importancia de la etiqueta <label>:

  • Conexión: Es vital conectar el texto con el input mediante el atributo for (en label) e id (en input).
  • Importancia: Mejora la accesibilidad y amplía el área de clic para el usuario.

2. Menús de Selección (Select y Datalist)

Vimos cómo gestionar listas de opciones:

  • <select>: Uso de multiple para varias elecciones y size para la altura visual.
  • <optgroup>: El profesor hizo hincapié en agrupar opciones para formularios largos.
  • <datalist>: La gran diferencia es que permite "sugerir" valores pero deja al usuario escribir algo nuevo.

3. Organización con Fieldset y Legend

⚠️ ENFASIS DEL PROFESOR: El uso de <fieldset> y <legend> no es solo por el recuadro gris. Es por semántica. Ayuda a los lectores de pantalla a saber que un grupo de preguntas pertenecen al mismo tema (ej: "Datos Personales").

4. Atributos de Control y Validación Nativa

Conceptos clave de validación antes de entrar en Regex:

  • Readonly vs Disabled: El readonly envía el valor al servidor; el disabled no envía nada.
  • Límites: Atributos min, max, minlength, maxlength y step.

5. Expresiones Regulares (El "Superpoder")

Utilizamos el atributo pattern para validaciones complejas:

  • Código Postal: [0-9]{5} (5 dígitos exactos).
  • Teléfono: [6-9][0-9]{8} (Empieza por 6, 7, 8 o 9 y sigue con 8 números).
  • NIF: [0-9]{8}[A-Za-z] (8 números y una letra).
  • Contraseña segura: (?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}
📌 NOTA IMPORTANTE: Siempre hay que añadir el atributo title. Es la única forma de que el usuario sepa por qué su dato no es válido según el pattern.

6. Ejercicio: Fecha Dinámica con JS

Finalizamos resolviendo cómo hacer que un calendario de reserva no permita elegir fechas pasadas:

<script>
document.getElementById('reserva').min = new Date().toISOString().split('T')[0];
</script>
